# FRONTEND VISION — ODONTO.IA

## Objetivo

Antes de qualquer nova funcionalidade, o foco atual do projeto é consolidar e elevar a qualidade visual de toda a aplicação.

O objetivo não é apenas corrigir telas específicas.

O objetivo é transformar o Odonto.IA em uma experiência visual única, consistente e premium.

---

# REGRA PRINCIPAL

Toda tela do sistema deve atingir o mesmo nível de qualidade visual do:

* Dashboard
* Tratamento

Essas duas telas são atualmente a referência oficial de UX/UI do projeto.

Qualquer tela que pareça inferior, mais simples ou desconectada dessas referências deve ser revisada.

---

# VISÃO DE PRODUTO

O Odonto.IA não deve transmitir a sensação de:

* ERP tradicional
* Sistema administrativo
* CRUD corporativo
* Software legado
* Aplicação genérica

O Odonto.IA deve transmitir:

* Tecnologia
* Sofisticação
* Modernidade
* Fluidez
* Clareza
* Inteligência operacional
* Produto premium

---

# REFERÊNCIAS DE EXPERIÊNCIA

Inspirar-se em princípios de produtos como:

* Linear
* Notion
* Attio
* Vercel Dashboard
* Stripe Dashboard

Não copiar layouts.

Copiar princípios.

---

# PRINCÍPIOS VISUAIS

## Hierarquia Forte

O usuário deve identificar instantaneamente:

* o que é mais importante
* o que é secundário
* qual ação deve ser executada

Nenhuma tela deve competir visualmente consigo mesma.

---

## Espaçamento Generoso

Priorizar:

* respiro visual
* agrupamento lógico
* separação clara de contexto

Evitar telas comprimidas.

Evitar excesso de informação por bloco.

---

## Baixa Carga Cognitiva

Cada tela deve responder rapidamente:

* Onde estou?
* O que estou vendo?
* O que posso fazer agora?

Sem exigir esforço mental desnecessário.

---

## Consistência

Componentes equivalentes devem possuir:

* mesma linguagem visual
* mesma hierarquia
* mesmos espaçamentos
* mesmos estados

O sistema inteiro deve parecer construído pela mesma equipe.

---

## Sensação Premium

Priorizar:

* refinamento
* clareza
* elegância
* simplicidade

Evitar:

* excesso de elementos
* excesso de cores
* excesso de bordas
* excesso de informações simultâneas

---

# MODO CLARO

O modo claro deve receber atenção especial.

Atualmente existem elementos com baixo contraste e legibilidade reduzida.

Garantir:

* contraste adequado
* leitura confortável
* separação clara entre seções
* acessibilidade visual

Nenhum elemento importante deve ficar difícil de visualizar no Light Mode.

---

# PADRÃO VISUAL OFICIAL

Dashboard e Tratamento representam o padrão visual oficial do projeto.

Ao criar ou modificar qualquer tela, comparar com essas referências.

Pergunta obrigatória:

"Esta tela parece ter sido criada pela mesma equipe que criou o Dashboard e o Tratamento?"

Se a resposta for não, a implementação não está concluída.

---

# OBJETIVO FINAL

Ao navegar entre:

* Dashboard
* Pacientes
* Perfil do Paciente
* Tratamento
* Ficha Clínica
* Agenda
* Orçamentos
* Financeiro
* Configurações

o usuário não deve perceber diferença de qualidade visual.

Todas as áreas devem transmitir:

* tecnologia
* sofisticação
* consistência
* identidade própria

O Odonto.IA deve parecer um único Workspace Clínico moderno, premium e cuidadosamente projetado.

Não apenas um conjunto de telas conectadas.
